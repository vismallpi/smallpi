# 2026-05-04 每日开发记录

## Amazon搜索控制面板开发迭代

### v1.4.6 - 修复基础问题
- 修复了参数传递问题：原来修改静态脚本文件传参导致参数丢失败，改成**命令行参数传递**，前端选的值总能正确传给后端
- 修复了log重复问题：原来每日监控脚本手动发log + print重写自动发log，导致每条log发两次。现在统一机制：**只用print重写自动发**，每条log只发一次，解决重复问题

### v1.5.0 - 新增log文件 + 在线查看功能
- 每个脚本运行都创建独立log文件（带RUN_ID时间戳命名）
- 每一行log都自动添加**北京时区时间戳**，方便追溯
- 前端每个section按钮旁边添加 **"View Latest Log"** 按钮，点击弹出弹窗显示最新log
- log目录已创建：`/root/.openclaw/workspace/Amazon/logs/`

### v1.5.1 - 截图和log统一管理
- 每次运行，截图都保存到对应run目录下，和log放在一起：
  ```
  logs/
  ├─ {RUN_ID}_manual_search.log
  └─ {RUN_ID}_screenshots/
       └─ *.png
  ```

### v1.5.2 - 弹窗中直接预览截图 + 超链接打开原图
- "View Latest Log" 弹窗中，log文字在上，**自动提取截图生成预览画廊**在下
- 每张截图显示预览，下方有超链接可以点击直接打开原图新标签
- 修复了页面title可编辑功能，现在标题编辑可以自动保存了
